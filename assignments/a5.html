Frequently Asked Questions

Q: Is this an individual assignment or a team assignment?

A: An individual assignment. You are free to collaborate with your partner but each student will submit his/her own report.

Q: Are we supposed to read the downloaded source code in order to make our attacks on the site we are attacking? Or should we try to do it without going through the source code? I'm not sure how to tell my attacks are working.

A: Two step process:

  1. First, perform attacks WITHOUT reading the source code! This is also known as "black box" testing.
  2. Then, download or get from your partner, the source code of the application. Peruse through it, perform a manual code review of it, and document for your findings.

If you find anything glaring, then go back to the application and attack it. This is also known as "white box" testing. For example, after you read the source code for the search function and you are suspicious that you can perform a SQL injection attack to delete a table (also given the database structure from your partner), go ahead an try it out :-)

Q: Is it necessary to use all the programs listed above? I'm having trouble running Burp Suite specifically. I was able to get a good amount of information from Paros and from looking at the source code, though.

A: No.

Q: I'm using a tool and it gave me back a URL under alerts. Does that mean it tried a SQL injection and worked?

A: Perhaps. It could also be a false positive. You have to check in the database (in this case, ask your partner), if it went through. However, this is something that you can document.

Q: If a tool doesn't give me back any alerts when spidering through something, does that mean that it hasn't found any, or that I don't know how to use the application?

A: If a tool doesn't return anything, it didn't find anything. But any good security practitioner would know that you cannot depend on a (especially just one) scanning tool.

Q: I can't find any problems with my partner's application. What should I do?

A: You can make security recommendations. I will find it hard to believe that you cannot find problems. Example: if you notice that your partner's web application uses the insecure methods, you can recommend to upgrade application to use more secure options.
